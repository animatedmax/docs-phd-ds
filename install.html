To create a Service Plan:

  1. Use a Web browser to open the Pivotal Ops Manager application. (This application is part of your Pivotal Cloud Foundry installation.)

  2. Click the Pivotal HD for Pivotal CF tile.

    The Pivotal HD configuration page opens.

  3. If it is not already selected, click Network Settings.

  4. Configure the following network settings for your Pivotal HD cluster plan:

    Network Settings
    Parameter Description Values
    Network Name The name of the network you want all Pivotal HD virtual machines to reside on. If the network is different than the network you configured in Ops Manager vSphere settings, you must ensure connectivity between the two networks so that the service can monitor the Pivotal HD virtual machines properly.
    IP Address Subnet A valid subnet in which to deploy virtual machines.

    Choose IP addresses that do not overlap with those you assigned to Ops Manager (??where) to ensure Pivotal HD virtual machines and Pivotal CF virtual machines are assigned unique IP addresses.

    Enter the subnet using CIDR notation. For example:

    10.10.10.10/16.

    Excluded IP Address Range List any IP addresses you wish to reserve. These addresses will not be used by by the Pivotal HD Service to create virtual machines.

    Separate multiple ranges with commas. For example:

    10.10.10.2-10.10.10.10, 10.10.10.200-10.10.10.254
    DNS Servers One or more IP Addresses of the Domain Name Servers. .
    Default Gateway IP Address IP address of the default gateway. Separate the IP addresses with a comma.

  5. Click Save.

  6. Click On-demand Service Plans.

    The On-demand Service Plans screen displays.

  7. Configure the following display options for your Pivotal HD cluster plans. This information displays in the Pivotal CF Console and Command-line Interface.

  8. Enter the following values for multi-tenant configuration. The values are informational only ??
    Multi-tenant configuration
    Parameter Description Values
    Amount Cost, in US dollars to create a Pivotal HD cluster instance from this Service Plan US dollars
    Unit of Measurement Unit of measurement used for billing, for example: Monthly, or weekly. Text
  9. Enter the following values to define your cluster instances:
    Cluster Instances
    Parameter Description Values
    Maximum Number of Instances Maximum number of Pivotal HD instances that users can create from this Service Plan. Integer, Leave this field blank if you do not wish to specify a maximum number of instances.
    Number of Pre-Created Instances Pivotal HD Service creates a reserved pool of Service Plan Instances to allocate to Pivotal CF Users. When an Service Plan Instance is allocated in Pivotal CF Elastic Runtime, the service back-fills the pool.

    The Service never pre-creates more instances than the configured Maximum Number of Instances.

    • Minimum: 1
    • Maximum: Maximum Number of Instances Created (configuration value, see above)
  10. Fill in the remaining fields to define the cluster plan. The cluster plan defines the nodes and components of the Pivotal HD clusters that Pivotal HD Service creates.

    For more information on configuring Pivotal HD clusters, see the Pivotal HD Documentation .
