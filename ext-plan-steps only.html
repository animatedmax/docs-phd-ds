1. Use a Web browser to open the Pivotal Ops Manager application. (This application is part of your Pivotal Cloud Foundry installation.)
  2. Click the Pivotal HD for Pivotal CF tile.
  3. Select External Service Plans.

    The External Service Plans screen displays.

  4. Click Add to create a new Service Plan.
  5. Configure the following options for your Service Plan. This information displays in the Pivotal CF Console and Command-line Interface:
    Informational Display
    Parameter Description Values
    Service Plan Name (Required) The name you want Pivotal CF users to see in the CLI and Developer Console. String
    Service Plan Feature Bullet 1 The details of the service plan you want Pivotal CF users to see. The values in these fields display to users in the Marketplace section of the Pivotal CF Developer Console. String
    Service Plan Feature Bullet 2
    Service Plan Feature Bullet 3
  6. Enter the following values regarding the cost of Pivotal HD clusters. These fields are for display only. The values in these fields display to users in the Marketplace section of the Pivotal CF Developer Console:
    Cost Information
    Parameter Description VAlues
    Amount (Required) The cost, if any, in US dollars to create an instance of this Service Plan.  For example: inputting 1000 will display $1000 Integer
    Unit of Measurement (Required) Unit of measurement used for billing.  For example: Monthly, or weekly. String
  7. Select the Pivotal HD Components whose information you want to include in the Service Plan. If you do not select a component, the related input fields on the page for that component are ignored. You can select any of the following components:
    Pivotal HD Components
    Parameter Description
    HDFS Controls whether to include information about the NameNode host and DFS port as well as the Web UI port.
    Yarn/MapReduce2 Controls whether to include information about the ResourceManager’s host and port, scheduler port, and the Web UI port, as well as the JobHistoryServer’s UI port.
    HAWQ Controls whether to include information about the HAWQ Master’s host and port as well as the database name.
    GemFire XD Controls whether to include information about the GemFire XD Locator’s host and port.
  8. Enter the following values for all relevant fields of the Pivotal HD software you want the Service Plan to broker access to. These values you enter in these fields are displayed in the Service Instance Dashboard Pivotal CF users can access when they create a Service Instance and/or in the VCAP_SERVICES environment variable Applications can access when they are bound to the Service Instance.

  9. (Optional) Repeat the steps above to create additional External Service Plans.
  10. Click Save to save the Service Plan(s).
